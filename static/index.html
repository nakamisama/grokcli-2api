概览

客户端 API Key

列表可随时 复制 完整 Key（升级前创建的旧 Key 若无完整值需重建）。 调用 /v1 时带 Authorization: Bearer sk-g2a-… 或 x-api-key。

名称前缀状态请求数创建时间

轮询策略

多账号按策略分配请求（账号完全对等，无主账号）。上游 401/429/5xx 会冷却并自动切换； 额度耗尽会自动移出轮询，直到你手动重新启用。 同一对话会粘性绑定账号（不因轮询中途切号打断多轮记忆）；仅失败时才 failover。

Grok 账号

独立运行，无需本地 Grok CLI： 点「设备码登录」（原生 OIDC）→ 用手机/本机浏览器打开验证链接并输入设备码。 也可直接粘贴 JWT / auth.json 到下方「导入」，勾选合并实现多账号轮询。 凭证保存在项目 data/auth.json；额度查询走上游 /v1/billing，耗尽后自动关闭轮询。
